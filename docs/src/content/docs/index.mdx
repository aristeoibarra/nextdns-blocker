## Why NextDNS Blocker?

NextDNS Blocker goes beyond simple domain blocking. It's a **behavioral tool** designed to help you build healthier digital habits through intelligent scheduling and friction-based protection.

<CardGrid stagger>
  <Card title="Per-Domain Scheduling" icon="sun">
    Configure unique availability hours for each domain. Block Reddit during work hours, allow it during lunch breaks.
  </Card>
  <Card title="Unblock Delays" icon="clock">
    Add friction against impulsive unblocking. Wait 30 minutes, 4 hours, or make domains permanently protected.
  </Card>
  <Card title="Panic Mode" icon="warning">
    Emergency lockdown that blocks all domains and hides dangerous commands. Perfect for crisis moments.
  </Card>
  <Card title="Cross-Platform" icon="laptop">
    Native support for macOS (launchd), Linux (cron), and Windows (Task Scheduler) with automatic synchronization.
  </Card>
</CardGrid>

## Quick Install

<Tabs>
  <TabItem label="Homebrew">
    ```bash
    brew tap aristeoibarra/tap
    brew install nextdns-blocker
    nextdns-blocker init
    ```
  </TabItem>
  <TabItem label="pip">
    ```bash
    pip install nextdns-blocker
    nextdns-blocker init
    ```
  </TabItem>
  <TabItem label="Docker">
    ```bash
    git clone https://github.com/aristeoibarra/nextdns-blocker.git
    cd nextdns-blocker
    cp .env.example .env && cp config.json.example config.json
    docker compose up -d
    ```
  </TabItem>

## How It Works

<div class="diagram-container">

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   config.json   │────▶│  nextdns-blocker │────▶│   NextDNS API   │
│   (schedules)   │     │   (sync engine)  │     │   (denylist)    │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                               │
                               ▼
                        ┌──────────────────┐
                        │    Watchdog      │
                        │  (auto-sync 2m)  │
                        └──────────────────┘
```

</div>

<div class="steps-list">
  <div class="step-item">
    <strong>Configure</strong> your domains and schedules in <code>config.json</code>
  </div>
  <div class="step-item">
    <strong>Sync</strong> evaluates each domain against the current time and your schedule
  </div>
  <div class="step-item">
    <strong>NextDNS API</strong> receives block/unblock commands for your denylist
  </div>
  <div class="step-item">
    <strong>Watchdog</strong> runs automatically every 2 minutes to enforce your rules
  </div>
</div>

## Example Configuration

<div class="terminal-window">
  <div class="terminal-header">
    <span class="terminal-dot terminal-dot--red"></span>
    <span class="terminal-dot terminal-dot--yellow"></span>
    <span class="terminal-dot terminal-dot--green"></span>
    <span class="terminal-title">config.json</span>
  </div>
  <div class="terminal-body">

```json
{
  "blocklist": [
    {
      "domain": "reddit.com",
      "description": "Social media - limited access",
      "unblock_delay": "30m",
      "schedule": {
        "available_hours": [
          {
            "days": ["monday", "tuesday", "wednesday", "thursday", "friday"],
            "time_ranges": [
              {"start": "12:00", "end": "13:00"},
              {"start": "18:00", "end": "22:00"}
            ]
          },
          {
            "days": ["saturday", "sunday"],
            "time_ranges": [
              {"start": "10:00", "end": "23:00"}
            ]
          }
        ]
      }
    }
  ]
}
```

  </div>
</div>

**This configuration:**

    Open access 6pm-10pm on weekdays
  </Card>

    30-minute wait to manually unblock
